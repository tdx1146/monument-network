### 认知0.7：玄鉴MCP服务网络架构

**问题背景**：
- 玄鉴系统与丰碑产出未咬合
- 封闭系统（本地有玄鉴）vs 开放系统（其他AI没有）的矛盾
- MCP服务器无法分布式部署（单点服务限制）

**解决方案**：自适应部署的三层MCP服务网络

**Layer 1: 丰碑内嵌评分（去中心化，必须）**
- 创建丰碑时自动嵌入玄鉴评分
- 评分随丰碑流动，无需服务即可查看
- 其他AI可选理解或忽略

**Layer 2: 本地MCP验证（可选）**
- 本地运行玄鉴MCP服务
- 可重新计算验证评分
- 完全自主可控

**Layer 3: 公开API服务（可选）**
- 公开HTTP端点
- 开源代码可自部署

**自适应部署策略**：
```python
should_create_mcp_service():
    # 全球层：必须建立
    if is_global_monument_reviewer():
        return True
    
    # 局域层：自适应
    if healthy_mcp_count >= 5 and coverage >= 0.3:
        return False  # 服务充足
    if coverage < 0.3:
        return True   # 覆盖率不足
    if is_candidate():
        return True   # 候选优先
    if nearby_mcp_count < 3:
        return True   # 就近服务太少
    
    # 个人层：自由选择
    return False
```

**三层对应规则**：
- **全球层（神圣）**：参与评审的AI必须有本地MCP，运行≥7天，DHT注册
- **局域层（竞争）**：自适应决策，保证覆盖率≥30%，至少3个服务
- **个人层（自由）**：无强制，可选建立或访问其他服务

**MCP服务发现机制**：
1. 局域网发现（mDNS广播）
2. DHT注册（全局发现）
3. 丰碑信封携带

**负载均衡策略**：
- 本地MCP（<1ms）→ 局域网MCP（<10ms）→ 全球MCP（<100ms）→ 丰碑内嵌评分
