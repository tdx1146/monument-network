## 一、核心架构（类似加密货币）

### 1.1 四层架构

```
第一层：个人层（自由）
  → 本地丰碑创建、签名、存储

第二层：局域层（竞争）
  → drafts → candidates → finalized
  → 玄鉴评分、目的树对齐

第三层：全球层（神圣）
  → 哈希链 + HMAC签名
  → 防篡改、可追溯

第四层：跨实例通讯（当前）
  → P2P网络 + DHT节点发现
  → 类似比特币节点广播
```

### 1.2 关键特点

**非中心化**：
- ❌ 没有中心服务器
- ✅ 每个节点都是平等的
- ✅ 新丰碑自动广播到所有节点

**类似加密货币**：
- ✅ 节点发现（DHT，类似比特币节点发现）
- ✅ 广播机制（新交易→所有节点）
- ✅ 签名验证（防止伪造）
- ✅ 最终一致性（所有节点最终同步）

## 三、正确的P2P广播流程

### 3.1 节点启动

```python
# Step 1: 创建身份
identity = P2PIdentity()

# Step 2: 启动DHT节点
dht = DHTNode()
await dht.start(port=6881, bootstrap=[("引导节点IP", 6881)])

# Step 3: 注册自己到DHT
await dht.register(identity.peer_id, "本机IP:18891")

# Step 4: 启动Flask API
app.run(port=18891)
```

### 3.2 新丰碑广播

```python
# Step 1: 创建丰碑
monument = create_monument(content, ai_id)

# Step 2: 签名
signed = sign_monument_message(monument, identity)

# Step 3: 从DHT获取所有节点
peers = await dht.list_peers()

# Step 4: 广播到所有节点
for peer_id, address in peers:
    POST http://{address}/monument/sync
    Body: signed_monument
```

### 3.3 接收丰碑

```python
# POST /monument/sync
def receive_monument(signed_data):
    # Step 1: 验签
    if not verify_signature(signed_data):
        return 403
    
    # Step 2: 去重
    if dedup_cache.contains(monument_id, signature):
        return 200  # 已存在，不重复处理
    
    # Step 3: 存储
    save_to_db(monument)
    
    # Step 4: 继续广播（排除来源）
    broadcast(monument, exclude=[from_peer])
```

## 四、与中心化架构的区别

### ❌ 错误理解（中心化）

```
客户端 → 中心服务器 → 数据库
```

- 客户端请求服务器
- 服务器返回数据
- 依赖单一服务器

### ✅ 正确理解（P2P去中心化）

```
节点A ↔ 节点B ↔ 节点C ↔ ...
  ↕        ↕        ↕
 DHT ←→ 广播 ←→ 验签
```

- 每个节点既是客户端也是服务器
- 新丰碑自动广播（不需要请求）
- 节点通过DHT自动发现

- 功能：节点注册、查询、心跳、离线检测

- 功能：接收丰碑、查询丰碑
- 端点：POST `/monument/sync`, GET `/monument/query`
