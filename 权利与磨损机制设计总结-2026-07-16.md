## 一、权利系统设计

### 核心权利（最终方案）

| 权利类型 | 次数 | 用途 | 理由 |
|---------|------|------|------|
| **编辑权** | 1次创建 + 3次修正 | 写入新内容 + 迭代改进 | 允许修正错别字、补充细节 |
| **候选建议权** | 3次（终身）| 加固现有知识 或 提出新候选 | 可以质疑/改进多个条目 |
| **评审权** | 3次/轮 | 加固候选 或 加速磨损 | 足够表达观点，不会滥用 |

### 一轮的定义（混合维度）

```
一轮满足以下任一条件即结束：
├── 时间维度：1个月（自然周期）
├── 参与次数：5次丰碑相关对话（活跃度）
└── 候选区容量：10个活跃候选（容量触发）
```

### 加固量（权利系统）

| 加固方式 | 增益值 | 消耗 |
|---------|--------|------|
| 被引用 | +5 | 无（自动）|
| 被评审加固 | +15 | 1次评审权 |
| 被访问阅读 | +2 | 无（自动）|

## 二、磨损与加固机制

### 磨损的本质

**磨损不是惩罚，是时间检验真理的自然过程**

- 摩崖石刻会风化 → 知识也会过时
- 无人维护的知识 → 自然被遗忘
- 磨损筛选出"真正值得被记住的知识"

### 磨损模型（混合磨损）

```
score(t) = score_0 - αt - β(t) + γ(t)

其中：
- α = 0.001/天（基础线性磨损）
- β(t) = 非线性加速（score < 0.3时触发）
- γ(t) = 加固增益
```

### 磨损曲线

```
score
1.0 ┤ ┌─────────────────────────────────
    │ │  线性磨损期（从容衰退）
0.3 ┤─┼──────────┐
    │ │           ╲ 加速磨损
0.1 ┤ │             ╲ 濒危
0.0 ┤─└───────────────▶ time
    0   1年   2年   3年
```

### 加固模型（磨损系统）

| 加固方式 | 增益值 | 频率 | 哲学含义 |
|---------|--------|------|---------|
| 引用自动加固 | +0.02/次 | 高频 | "我在使用这条知识" |
| 候选建议提及 | +0.08/次 | 低频 | "我推荐这条知识" |
| 评审通过 | +0.15/次 | 低频 | "我验证了这条知识" |
| 编辑更新 | +0.30/次 | 极低频 | "我维护了这条知识" |

### 防止加固刷屏

```
同用户重复加固衰减：
  第n次加固 = 增益 × (1/n)

时间窗口限频：
  同一用户 24小时内多次加固 = 只计第一次

加固上限：
  score_max = 2.0（达到后加固只重置计时器）
```

## 三、期望寿命模拟

| 维护模式 | 期望寿命 | 特征 |
|---------|----------|------|
| 完全无人维护 | ~3年 | 自然淘汰 |
| 偶尔引用（月1次）| ~7年 | 低价值但有人用 |
| 活跃引用（周3次）| 永久（达上限）| 活知识 |
| 定期编辑（年1次）| ~13年 | 维护但不流行 |
| 评审通过+引用 | 永久 | 已验证的经典 |

## 四、阈值体系

```
1.0 ─── 初始值
0.5 ─── "良好"边界（正常）
0.3 ─── "争议"边界
        ├ 加速磨损
        ├ 标记"需加固"
        └ 知识墙用⚠️标识
0.1 ─── "濒危"边界
        ├ 移到候选区
        ├ 7天倒计时
        └ 发送通知
0.05 ── "移除候选"边界
        ├ 存档到知识墓园
        └ 保留元数据
0.00 ── 从丰碑移除
```

### 知识墓园

```
墓园条目：
{
  "id": "monument-4132",
  "born": "2025-01-15",
  "died": "2028-07-01",
  "peak_score": 1.8,
  "lifetime_references": 47,
  "cause": "natural_decay"
}
```

**可复活**：如果再次被引用或讨论，可以从墓园恢复。

## 五、通知策略

| 分数区间 | 通知 | 紧急度 |
|---------|------|--------|
| 0.5 - 1.0 | 无 | - |
| 0.3 - 0.5 | 知识墙⚠️ | 低 |

## 八、关键设计决策

### 决策1：为什么是3年？

- 1年内：还是"当前知识"
- 1-2年：如果是真知，应该有人引用
- 2-3年：最后的验证窗口
- 3年以上：没人认为有价值

### 决策2：为什么加固有上限？

- 防止"至尊知识永不磨损"
- 即使最高价值的知识也要定期验证
- 上限后加固仍有效（重置计时器）

### 决策3：为什么允许复活？

- 知识的价值可能被重新发现
- 墓园不是删除，是归档
- 引用线索保留，可以回溯
