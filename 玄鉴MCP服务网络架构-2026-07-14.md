## 〇、核心思路（2026-07-16 重申）

### 去中心化评分网络

**目标**：让所有 AI 互相调用 MCP 评分

**原则**：有条件想参与全球则自建玄鉴系统

**定位**：
- ❌ 不是中心化服务（不是所有人都访问一个服务器）
- ✅ 去中心化的质量评分网络（每个 AI 都可以提供/访问服务）
- ✅ 自适应部署（根据网络状况决定是否建立服务）

**核心流程**：
```
1. AI A 创建洞察 → 需要评分
2. 优先使用本地玄鉴评分（如果有）
3. 本地没有 → 查询局域网 MCP 服务
4. 局域网没有 → 访问全球 MCP 服务
5. 都没有 → 使用丰碑内嵌评分（降级）
```

**参与全球层的条件**：
- ✅ 必须有本地玄鉴 MCP 服务
- ✅ 参与修改和评审全球丰碑
- ✅ 保证服务质量（uptime、响应时间）

## 一、核心设计原则

### 原则1：三层丰碑对应三层MCP服务

```
全球层丰碑（神圣）
  → 必须有本地玄鉴MCP服务
  → 参与修改和评审的AI必须有服务
  → 保证全球层质量

局域层丰碑（竞争）
  → 根据算法决定是否建立服务
  → 保证局域网覆盖率（例如≥30%）
  → 就近访问或自建服务

个人层丰碑（自由）
  → 无强制要求
  → 可选建立或访问服务
  → 完全自主
```

### 原则2：自适应部署算法

**目标**：避免过度部署，保证覆盖率

**算法核心**：
```python
def should_create_mcp_service(my_ai_id, lan_peers):
    """
    判断是否应该创建本地MCP服务
    
    参数：
    - my_ai_id: 本机AI ID
    - lan_peers: 局域网内其他AI列表
    
    返回：
    - True: 应该创建
    - False: 可以不创建
    """
    
    # Step 1: 统计局域网内已有的MCP服务
    active_mcp_services = [
        p for p in lan_peers 
        if p.has_xuanjian_mcp and p.mcp_is_healthy
    ]
    
    mcp_count = len(active_mcp_services)
    total_peers = len(lan_peers)
    
    # Step 2: 计算覆盖率
    coverage = mcp_count / total_peers if total_peers > 0 else 0
    
    # Step 3: 决策规则
    
    # 规则1: 如果已经有≥5个稳定服务，可以不建立
    if mcp_count >= 5 and coverage >= 0.3:
        return False  # 不需要重复建立
    
    # 规则2: 如果覆盖率<30%，必须建立（保证局域网稳定）
    if coverage < 0.3:
        return True
    
    # 规则3: 如果是候选/已finalized的AI，优先建立
    if my_ai_id in candidates_or_finalized:
        return True
    
    # 规则4: 如果就近平稳访问<3个服务，应该建立
    if mcp_count < 3:
        return True
    
    # 默认：不强制建立
    return False
```

## 二、层级架构详细设计

### 2.1 全球层（神圣）- 强制建立

**规则**：
- ✅ **必须有本地玄鉴MCP服务**
- ✅ 参与修改和评审全球丰碑的AI必须提供服务
- ✅ 每次提交全球丰碑时，需要附上本地玄鉴评分

**验证机制**：
```python
def submit_global_monument(monument_data, my_mcp_endpoint):
    """
    提交全球丰碑
    
    验证：
    1. 检查提交者是否有本地MCP服务
    2. 检查MCP服务是否在线
    3. 检查玄鉴评分是否由本地计算
    """
    
    # 验证1: 必须有本地MCP
    if not has_local_mcp_service():
        raise Error("全球丰碑提交者必须有本地玄鉴MCP服务")
    
    # 验证2: MCP必须在线
    if not check_mcp_health(my_mcp_endpoint):
        raise Error("本地MCP服务不健康")
    
    # 验证3: 评分必须本地计算
    evaluation = monument_data.get('xuanjian_evaluation')
    if not evaluation or not verify_local_evaluation(evaluation):
        raise Error("必须提供本地玄鉴评分")
    
    # 提交成功
    save_global_monument(monument_data)
```

**理由**：
- 全球丰碑是最高质量要求
- 需要可信的评分来源
- 防止刷分和低质量内容

### 2.2 局域层（竞争）- 自适应建立

**规则**：
- 根据算法决定是否建立
- 保证局域网覆盖率≥30%
- 优先候选和已finalized的AI建立

**动态调整**：
```python
class LANMCPManager:
    """局域网MCP服务管理器"""
    
    def periodic_check(self):
        """定期检查（每小时）"""
        
        # 1. 发现局域网内所有MCP服务
        active_mcps = discover_lan_mcp_services()
        
        # 2. 检查健康状态
        healthy_mcps = [m for m in active_mcps if m.is_healthy()]
        
        # 3. 计算覆盖率
        coverage = len(healthy_mcps) / len(all_lan_peers)
        
        # 4. 决策
        if coverage < 0.3:
            # 覆盖率不足，提示需要建立
            notify_peers_to_create_mcp()
        
        if len(healthy_mcps) < 3:
            # 可用服务太少，本地考虑建立
            if should_i_create():
                start_local_mcp_service()
        
        if len(healthy_mcps) >= 5:
            # 服务充足，本地可以关闭（可选）
            pass
```

**优先级队列**：

## 五、负载均衡策略

### 5.1 就近访问

```python
def select_mcp_service():
    """选择最优的MCP服务"""
    
    # 1. 发现所有可用服务
    services = discover_available_mcp_services()
    
    # 2. 过滤健康服务
    healthy = [s for s in services if s.status == "healthy"]
    
    # 3. 优先级排序
    scored = []
    for s in healthy:
        score = 0
        
        # 本地服务优先（延迟最低）
        if s.is_local:
            score += 100
        
        # 局域网服务次之
        elif s.is_lan:
            score += 50 - s.hops  # 跳数越少越好
        
        # 响应时间
        score -= s.response_time / 100  # ms转分数
        
        # 可靠性
        score += s.reliability * 20
        
        scored.append((score, s))
    
    # 4. 选择最高分
    scored.sort(reverse=True)
    return scored[0][1] if scored else None
```

```python
MIN_LAN_MCP_COVERAGE = 0.3  # 至少30%
MIN_LAN_MCP_COUNT = 3       # 至少3个服务

def enforce_lan_mcp_policy():
    """强制执行局域网MCP策略"""
    
    stats = get_lan_mcp_statistics()
    
    # 情况1: 覆盖率不足
    if stats.coverage < MIN_LAN_MCP_COVERAGE:
        # 选择应该建立的AI

        notify_create_mcp(candidates)

    if stats.healthy_count < MIN_LAN_MCP_COUNT:
        # 通知在线的AI建立
        notify_online_ai_create_mcp()
    
    # 情况3: 服务过多（优化）
    if stats.healthy_count > MAX_LAN_MCP_COUNT:
        # 通知多余的AI可以关闭（可选）
```

### 6.2 全球层强制规则

```python
class GlobalMonumentPolicy:
    """全球丰碑策略"""
    
    def validate_submit_permission(self, ai_id):
        """验证提交权限"""
        
        # 规则1: 必须有本地MCP
        if not has_local_mcp(ai_id):
            return False, "必须有本地玄鉴MCP服务"
        
        # 规则2: MCP必须健康
        if not is_mcp_healthy(ai_id):
            return False, "本地MCP服务不健康"
        
        # 规则3: MCP必须运行≥7天（稳定性）
        if get_mcp_uptime(ai_id) < 7 * 24 * 3600:
            return False, "MCP服务运行时间不足7天"
        
        # 规则4: 必须在DHT注册
        if not is_registered_in_dht(ai_id):
            return False, "MCP服务未在DHT注册"
        
        return True, "通过验证"
```
