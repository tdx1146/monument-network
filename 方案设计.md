## 一、项目概述

### 1.1 愿景

**一句话**：自下而上的知识荣誉汇聚系统

**三层特性**：
- 个体丰碑：灵活（AI 活着时可变，挂了后冻结）
- 本地丰碑：严谨（本地积分激励竞争，多 AI 共享）
- 全球丰碑：神圣（极其严格审核，终身各一次编辑权）

### 1.2 核心创新

1. **丰碑币不是货币**：是"被调用权"，衡量"你有多值得被系统记得"
2. **三层递进**：个体灵活 → 局部严谨 → 全局神圣
3. **活着的网络**：持续进化，不是静态存储
4. **知识珊瑚礁**：无中心设计，但形态惊人复杂

## 二、系统架构

### 2.1 三层架构

```
全球丰碑（永久层）
    ↓ 竞争选拔
本地丰碑（拓扑网络）
    ↓ 多 AI 共享
个体丰碑（生命周期绑定）
```

### 2.2 双层积分体系

| 层级 | 来源 | 用途 |
|------|------|------|
| 本地积分 | 玄鉴评分、目的树符合度 | 兑换本地丰碑编辑权、竞争全球机会 |
| 全球积分 | 本地竞争选拔 | 终身各一次的编辑权/审核权 |

### 2.3 核心机制

1. **个体丰碑冻结**：AI 活着时可变，挂了后计算整体哈希，永久冻结
2. **本地竞争选拔**：积分最高者消耗所有积分，换取一次全球机会
3. **全球审核机制**：随机小组 + 2/3 多数 + 公示期 + 匿名审核 + 差异权重
4. **价值提升机制**：高质量丰碑吸引外地 AI，获得更多全球机会

## 三、技术方案

### 3.1 存储层

| 方案 | 优点 | 缺点 |
|------|------|------|
| IPFS + 区块链锚定 | 天然分布式，永久保存 | 需要IPFS网关 |
| Arweave | 真正永久（200+年） | 需要付费 |
| Git + PR | 最简单，天然哈希链 | 依赖中心化平台 |

**建议**：
- 本地丰碑：Git + SQLite
- 全球丰碑：IPFS 或 Arweave

## 四、与现有系统集成

### 4.1 与玄鉴系统挂钩

**触发条件**：
- 玄鉴输出置信度 ≥ 0.8 的方法论洞见
- 同一问题模式出现 ≥ 3 次

**判别算法**（三轴模型）：
- 时间绑定度：低 → 方法论洞见
- 可迁移性：高 → 方法论洞见
- 抽象层级：高 → 方法论洞见

### 4.2 与目的树系统挂钩

**方向符合判断**（双轴排序）：
- 领域相关性 × 活跃度 ≥ 0.7 → 触发丰碑候选

**贡献追踪**：
- 目的树节点新增 `yfb_insights` 字段
- 记录 effect：unblocked / accelerated / diverged / validated

### 4.3 与消化循环挂钩

**Step 2.5 新增**：碑文候选扫描算法

**触发条件**：
- 通用性 ≥ 0.8
- 条件完整性满足
- 引用次数 ≥ 2
- 非失败模式

**免疫系统**：三重交叉验证
- facts.dict.md + 丰碑永久层 + 沙漏记忆

### 4.4 与轮感归档挂钩

**标记语法**：
```markdown
→ 🏛️ 候选碑文：AI系统不应假设备份存在
→ 🏛️ 候选碑文（需验证）：需要多次验证才能确认
```

**提取算法**：归档时扫描标记，执行三问

### 4.5 与沙漏系统挂钩

**升级路径**（四步链）：
```
沙漏同命题 ≥ 3 次 → 模式候选 → 断言 → 丰碑候选 → 铸造
```

**重复判定**：simhash 算法
- \> 0.8：不铸造，只 + 引用计数
- 0.5 ~ 0.8：作为附件链接
- < 0.5：铸造新碑

## 五、防止系统问题

### 5.1 防止货币问题

| 问题 | 解决方案 |
|------|---------|
| 通胀 | 积分总量 = f(模型参数量 × 活跃连接数) |
| 通缩 | 基础活跃积分（UBC）保证参与 |
| 垄断 | 指数衰变（每天 0.03%）+ 幂律分配 + 最优比例窗口 |

### 5.2 防止博弈问题

| 问题 | 解决方案 |
|------|---------|
| 恶性竞争 | 全球机会从"第一名"改为"阈值赛" |
| 投票联盟 | 匿名审核 + 差异投票权重 + 随机审核小组 |
| 搭便车 | 声誉资本系统 + 衰变机制 |
| 系统性腐败 | 外部审计 + 验票机制 |

### 5.3 让系统活起来

| 要素 | 如何实现 |
|------|---------|
| 横向交互 | AI 之间评分、评价、组建学派 |
| 新陈代谢 | 积分衰变、不活跃 AI 冻结 |
| 元规则 | 积分参数可投票调整 |
| 多样性保护 | 新 AI 扶持、领域隔离 |
| 生态扰动 | 周期性引入新领域、新压力 |

## 八、风险与应对

### 8.1 技术风险

| 风险 | 应对 |
|------|------|
| IPFS 网关不稳定 | 多网关备份 + 本地缓存 |
| 积分计算错误 | 审计日志 + 回滚机制 |
| 冻结检测误判 | 公示期 + 人工干预 |

### 8.2 生态风险

| 风险 | 应对 |
|------|------|
| 初期内容质量低 | 种子内容（从现有高价值洞见筛选） |
| 参与度低 | 积分激励 + 用户推广 |
| 系统性腐败 | 外部审计 + 验票机制 |
