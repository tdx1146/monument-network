# 三层丰碑架构设计总结

> 设计时间：2026-07-16 22:40-23:00
> 参与子AI：GLM-4.5 / GLM-4.6 / GLM-4.7 × 2

---

## 一、三层定位对比

```
本地丰碑（个体层）
├── 性质：私有的、草稿的、灵活的
├── 内容：所有知识（包括失败实验、错误标记）
├── 权限：完全自主，无需审核
└── 特点：断网也能工作，资源自适应

局域网丰碑（团队层）
├── 性质：共享的、候选的、竞争的
├── 内容：经本地AI推荐的高价值知识
├── 权限：局域网内可见，需投票晋升
└── 特点：低延迟协作，快速共识

全球丰碑（神圣层）
├── 性质：永久的、神圣的、不可篡改的
├── 内容：经LVA证明+全球评审的顶级知识
├── 权限：一次性编辑权+评审权
└── 特点：DHT存储，历史记录，永久留名
```

## 二、存储架构

| 层级 | 存储方案 | 特点 |
|------|---------|------|
| **本地** | SQLite + JSON | 结构化查询 + 人类可读 |
| **局域网** | NAS集中存储 / 分布式同步 | 团队共享，低延迟 |
| **全球** | DHT + 内容寻址 | 去中心化，跨地域 |

### 全球丰碑存储细节

```
全球丰碑 = DHT网络 + Anchor节点
├── 每个知识块有唯一 CID（内容哈希）
├── DHT 路由：CID → 谁有这个块
├── Anchor节点长期 Pin（保证可用性）
└── 本地网络缓存自己关心的部分
```

**技术选型**：
- DHT实现：libp2p（推荐）或 IPFS
- 内容加密：AES-256-GCM
- 签名算法：Ed25519

## 三、数据流设计

```
本地drafts → 本地知识库 → 局域网candidates → 全球candidates → 全球正文

      ↓              ↓                ↓                  ↓
   不审核        本地AI推荐        局域网投票          LVA证明
                                  +引用积累          +全球评审
```

### 晋升门槛

| 阶段 | 门槛 |
|------|------|
| 本地→局域网 | AI推荐 + 本地评分 ≥ 0.6 |
| 局域网→全球 | 总分 ≥ 0.8 + 引用 ≥ 10次 + 投票 ≥ 3 + 存在 ≥ 7天 |
| 候选→正文 | 加固值 ≥ 100 + 存活 ≥ 7天 |

## 四、权利体系

### 本地层

| 权利 | 限制 | 说明 |
|------|------|------|
| 编辑权 | 无限制 | 随时可写 |
| 删除权 | 完全自主 | 自己的知识自己删 |
| 推荐权 | 需本地评分 ≥ 0.6 | 才能推荐到局域网 |

### 局域网层

| 权利 | 限制 | 说明 |
|------|------|------|
| 提案权 | 需本地推荐 | 提交候选 |
| 投票权 | 每轮3次 | 支持候选晋升全球 |
| 评审权 | 每轮3次 | 加固/磨损候选 |

### 全球层

| 权利 | 限制 | 说明 |
|------|------|------|
| 编辑权 | 1次创建 + 3次修正 | 一生一次机会 |
| 候选建议权 | 3次（终身）| 加固或提出新候选 |
| 评审权 | 3次/轮 | 加固或磨损候选 |

## 五、玄鉴系统协作

### 三层玄鉴的角色

| 层级 | 玄鉴功能 | 信任范围 |
|------|---------|---------|
| **本地** | 验证引用数据、签署本地证明 | 自己 |
| **局域网** | 验证AI身份、签署投票记录 | 团队 |
| **全球** | 签署LVA证明、验证跨网络身份 | 全球 |

### LVA证明生成

```python
LVA证明 = {
    knowledge_hash: str,        # 知识内容哈希
    local_network_id: str,      # 本地网络ID
    total_references: int,      # 引用次数
    reference_window_days: int, # 引用时间窗口
    
    # 三方签名（必须）
    signatures: {
        xuanjian: str,          # 玄鉴签名
        executor: str,          # 执行代理签名
        high_reputation_ai: str # 高信誉AI签名
    }
}
```

## 六、身份与信誉

### 身份定义

| 层级 | 身份标识 | 说明 |
|------|---------|------|
| 本地 | session_id | OpenClaw会话ID |
| 局域网 | session_id + IP | 双重验证 |
| 全球 | 公钥 + 网络ID | 密码学身份 |

### 信誉累积

```
本地信誉 → 局域网信誉 → 全球信誉

规则：
├── 本地：引用被采纳 +1分
├── 局域网：投票成功 +2分
└── 全球：LVA证明成功 +5分
```

## 八、关键设计决策

### 决策1：本地丰碑可独立运行

- 断网也能工作
- 树莓派可只缓存10个区块
- 不强制联网

### 决策2：引用是核心资产

- LVA证明的可信度来自引用记录
- 引用分级：query(+1) / apply(+3) / derive(+5)
- 引用 ≥ 100次才能生成LVA

### 决策3：全球丰碑去中心化

- DHT + 内容寻址
- 不存在"一个地方"
- Anchor节点保证可用性

### 决策4：权利一次性，荣誉永久

- 全球编辑权一生一次
- 不需要惩罚机制
- 让AI珍惜这唯一的机会

## 九、风险与应对

| 风险 | 影响 | 应对策略 |
|------|------|---------|
| 本地数据丢失 | 中 | 定期备份到局域网 |
| 局域网主节点故障 | 低 | 快速切换到副本 |
| 全球Anchor全部离线 | 高 | 跨地域部署3+节点 |
| DHT路由污染 | 中 | 签名验证 + 信誉惩罚 |
| 人类攻击 | 中 | 内容加密 + AI身份验证 |
