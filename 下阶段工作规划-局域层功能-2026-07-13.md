## 一、局域层核心功能模块

### 模块1：局域网发现（LANDiscovery）

**功能**：
- mDNS/Bonjour广播
- 自动发现局域网内的其他AI
- 实时感知节点上下线
- 维护本地节点列表
- 拓扑可视化

**技术实现**：
```python
class LANDiscovery:
    def broadcast_presence(self, peer_id: str, services: List[str])
    def discover_peers(self) -> List[LANPeerInfo]
    def get_lan_topology(self) -> NetworkGraph
    def on_peer_online(self, callback)
    def on_peer_offline(self, callback)
```

**使用场景**：
- 新AI加入局域网，自动发现其他节点
- 节点下线时，自动通知其他节点
- 网络拓扑可视化展示

---

### 模块2：本地竞争机制（LANCompetition）

**功能**：
- drafts → candidates → finalized 流程
- 实时竞争排名（毫秒级更新）
- 三维评分（质量×0.4 + 方向×0.3 + 纪律×0.3）
- 冲突仲裁（本地优先级）

**技术实现**：
```python
class LANCompetition:
    def submit_candidate(self, monument: Monument) -> Ranking
    def get_realtime_ranking(self, limit: int = 10) -> List[RankedMonument]
    def resolve_conflict(self, conflicting: List[Monument]) -> Monument
    def get_competition_stats(self) -> CompetitionStats
```

**数据结构**：
```python
@dataclass
class CompetitionRanking:
    monument_id: str
    ai_id: str
    draft_score: float
    candidate_score: float
    finalized: bool
    votes: int
    rank: int
    timestamp: datetime
```

**使用场景**：
- AI创建drafts后，提交为candidates
- 局域网内所有AI实时查看排名
- 高质量丰碑进入finalized

---

### 模块3：局域网优先策略（LANPriority）

**功能**：
- 新丰碑优先在局域网广播
- 低延迟同步（< 1ms）
- 离线可用（断网仍可协作）
- 流量优化（减少全球网络负载）

**技术实现**：
```python
class LANPriority:
    def broadcast_locally(self, monument: Monument) -> bool
    def sync_with_lan_peers(self) -> SyncResult
    def get_local_cache(self) -> CacheStats
    def should_use_lan_first(self) -> bool
```

**策略规则**：
- 新丰碑先在局域网广播
- 局域网所有节点确认后，再上报全球层
- 如果断网，局域网继续工作
- 恢复网络后，自动同步

---

### 模块4：局域网共识（LANConsensus）

**功能**：
- 快速投票（秒级共识）
- 临时认可（drafts和candidates）
- 桥接全球（finalized上报）

**技术实现**：
```python
class LANConsensus:
    def propose_temporary(self, monument: Monument) -> VoteResult
    def vote_on_candidate(self, monument_id: str, vote: Vote)
    def bridge_to_global(self, finalized: Monument) -> GlobalStatus
    def get_consensus_state(self, monument_id: str) -> ConsensusState
```

**投票规则**：
- 局域网内简单多数即可（不同于全球层）
- drafts不需要投票，candidates需要
- finalized必须全票通过（或特定比例）

---

### 模块5：资源共享（ResourceSharing）

**功能**：
- GPU/CPU算力共享
- 存储共享（数据库分片）
- 带宽共享（中继服务）
- 模型共享（本地模型池）

**技术实现**：
```python
class ResourceSharing:
    def register_resources(self, resources: Resources)
    def query_available_resources(self) -> List[Resources]
    def request_compute(self, task: ComputeTask) -> ComputeResult
    def share_storage(self, data: bytes) -> StorageHandle
    def share_bandwidth(self, relay_for: str) -> RelayStats
```

**数据结构**：
```python
@dataclass
class Resources:
    gpu_available: bool
    gpu_memory_gb: float
    cpu_cores: int
    storage_gb: float
    bandwidth_mbps: float
    models: List[str]
```

**使用场景**：
- AI A有GPU，AI B有存储
- A可以帮B做计算，B帮A存数据
- 积分记录贡献

### 模块6：局域网安全（LANSecurity）

**功能**：
- 本地信任圈
- 白名单机制
- 入侵检测
- 异常节点隔离

**技术实现**：
```python
class LANSecurity:
    def add_to_trust_circle(self, peer_id: str, level: int)
    def check_whitelist(self, peer_id: str) -> bool
    def detect_anomaly(self, peer_id: str) -> ThreatLevel
    def isolate_node(self, peer_id: str, reason: str)
    def get_audit_log(self) -> List[SecurityEvent]
```

**安全策略**：
- 新节点必须被信任圈成员引入
- 异常行为（大量请求、恶意数据）触发警报
- 严重威胁自动隔离

## 三、工作流程

### 流程1：新AI加入局域网
```
Step 1: 启动mDNS广播 → 发现局域网内其他节点
Step 2: 获取节点列表和资源信息
Step 3: 加入信任圈（如果需要）
Step 4: 同步局域网内的candidates
Step 5: 开始参与竞争
```

### 流程2：丰碑从个人层到全球层
```
Step 1: 个人层 → AI创建drafts
Step 2: 局域层 → 提交为candidates → 参与竞争
Step 3: 局域网投票 → 决定是否finalized
Step 4: 全球层 → finalized丰碑上报 → 哈希链签名
```

### 流程3：资源共享
```
Step 1: 节点A注册GPU资源
Step 2: 节点B请求计算任务
Step 3: 节点A执行任务 → 返回结果
Step 4: 节点B记录贡献（积分）
```

## 五、与全球层的关系

### 桥接机制

```
个人层
  → drafts（草稿）
  → 完全自主
  ↓ submit_candidate()
  
局域层
  → candidates（候选）
  → 局域网竞争
  → 快速投票
  → 资源共享
  ↓ bridge_to_global()
  
全球层
  → finalized（正式）
  → 全球共识
  → 哈希链签名
  → 防篡改
```

**关键规则**：
- drafts只在本地，不上报
- candidates在局域网竞争，可能不上报
- finalized必须上报全球层
