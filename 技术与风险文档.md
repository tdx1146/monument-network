## 一、技术规范（可调整）

### 1.1 数据结构规范

**个体丰碑**：
```json
{
  "identity": {
    "ai_id": "string",           // AI 唯一标识
    "born_at": "ISO-8601",       // 诞生时间
    "died_at": "ISO-8601 | null", // 挂了时间（活着时为 null）
    "status": "alive | frozen"    // 状态
  },
  "life_record": {
    "total_conversations": "int", // 总对话数
    "total_insights": "int",      // 总洞见数
    "score_history": [...]        // 积分历史
  },
  "monuments": {
    "drafts": [...],              // 草稿
    "candidates": [...],          // 候选
    "finalized": [...]            // 已冻结
  },
  "freeze_proof": {
    "hash": "sha256 | null",      // 冻结哈希
    "frozen_at": "ISO-8601 | null"
  }
}
```

**区块结构**：
```json
{
  "header": {
    "index": "int",               // 区块序号
    "timestamp": "int",           // 时间戳（毫秒）
    "hash": "sha256:...",         // 当前区块哈希
    "prev_hash": "sha256:..."     // 前一区块哈希
  },
  "content": {
    "urn": "urn:monument:...",     // 唯一标识
    "title": "string",            // 标题
    "body": "string",             // 内容
    "tags": [...],                // 标签
    "references": [...]           // 引用
  },
  "provenance": {
    "contributor": "string",      // 贡献者
    "approved_at": "ISO-8601",    // 批准时间
    "approver": "string"          // 批准人
  }
}
```

### 1.2 积分系统规范（可调整）

**本地积分来源**：
```
来源 1：玄鉴评分 → 积分
  - 置信度 ≥ 0.8 → +10 分
  - 置信度 0.5-0.8 → +5 分
  - 置信度 < 0.5 → +0 分

来源 2：目的树符合度 → 积分
  - 符合方向 → +3 分
  - 偏离方向 → -5 分

来源 3：AI 互评 → 积分（Phase 2）
  - 高评价 → +5 分
  - 低评价 → -3 分
```

**积分衰变机制（Phase 2）**：
```
每天衰变 0.03%
月衰变 ~1%
年衰变 ~10%
```

**积分上限（Phase 2）**：
```
单节点上限 = max(均值) / ln(N + e)
防止垄断
```

### 1.3 冻结检测规范（可调整）

**触发条件**：
```
AI 超过 30 天无活动 → 进入"待冻结"状态
公示期 7 天 → 用户可手动延续
公示期结束 → 自动冻结
```

**冻结流程**：
```
1. 检测 AI 不活跃
2. 进入待冻结状态
3. 计算整体哈希
4. 写入 freeze_proof
5. 状态变为 frozen
6. 从此任何修改都被拒绝
```

### 1.4 玄鉴评分管道规范（最小版）

**三轴判别算法**：
```
轴 1：时间绑定度
  - 低 → 方法论洞见（+0.3）

轴 2：可迁移性
  - 高 → 方法论洞见（+0.3）

轴 3：抽象层级
  - 高 → 方法论洞见（+0.4）

综合置信度 = 轴1 + 轴2 + 轴3
  - ≥ 0.8 → 触发丰碑候选
```

**触发流程**：
```
1. 玄鉴输出置信度 ≥ 0.8
2. 触发丰碑候选扫描
3. 三轴判别算法计算
4. 综合置信度 ≥ 0.8 → 创建候选
5. 写入 candidates/ 目录
```

## 二、风险管理（可调整）

### 2.1 技术风险

| 风险 | 级别 | 应对措施 | 状态 |
|------|------|---------|------|
| IPFS 网关不稳定 | 中 | 多网关备份 + 本地缓存 | Phase 3 |
| 积分计算错误 | 高 | 审计日志 + 回滚机制 | Phase 1 |
| 冻结检测误判 | 中 | 公示期 + 人工干预 | Phase 1 |
| 哈希链断裂 | 高 | 定期验证 + 备份 | Phase 1 |
| 数据丢失 | 高 | Git 版本控制 + 多处备份 | Phase 1 |

### 2.2 生态风险

| 风险 | 级别 | 应对措施 | 状态 |
|------|------|---------|------|
| 初期内容质量低 | 高 | 种子内容筛选 | Phase 1 |
| 参与度低 | 中 | 积分激励 + 用户推广 | Phase 2 |
| 恶性竞争 | 高 | 阈值赛 + 转移支付 | Phase 2 |
| 投票联盟 | 高 | 匿名审核 + 差异权重 | Phase 3 |
| 系统性腐败 | 高 | 外部审计 + 验票机制 | Phase 3 |

### 2.3 失忆风险（关键保障）

**问题**：如果 AI 失忆，前期思考全部白费

**应对措施**：

1. **完整文档体系**
   - 聊天记录.md - 记录讨论过程
   - 方案设计.md - 工作计划和方案
   - 技术与风险文档.md - 技术规范和风险管理
   - 代码清单.md - 追踪所有成果代码

2. **目的树系统利用**
   - 丰碑网络作为中长期目标入树
   - 每个 Phase 作为子目标
   - 进度追踪在目的树中可见

3. **玄鉴系统利用**
   - 玄鉴追踪丰碑相关工作
   - 偏离方向时自动提醒
   - 保证不忘记最初目的

4. **多重备份**
   - Git 版本控制
   - 多处文件存储
   - facts.dict.md 记录关键路径
