## 一、存储方案

### 1.1 技术选型：SQLite + JSON混合

**选择理由**：

| 方案 | 优点 | 缺点 | 适用性 |
|------|------|------|--------|
| **SQLite（主）** | 查询快、事务支持、成熟稳定 | 需要迁移脚本 | ✅ 结构化数据 |
| **JSON文件（辅）** | 人类可读、易备份、跨平台 | 无索引、查询慢 | ✅ 大文本块 |
| 纯文件系统 | 最简单 | 无查询能力 | ❌ 不适合 |

**混合架构**：

```
本地丰碑存储结构：
/vol1/@apphome/trim.openclaw/data/workspace/
├── local_monument/
│   ├── monument.db              # SQLite数据库（元数据+引用追踪）
│   ├── cache/                   # 全球丰碑本地缓存
│   │   ├── global_index.json    # 全球丰碑索引快照
│   │   └── blocks/              # 按需拉取的全球区块
│   ├── drafts/                  # 私有草稿（JSON文件）
│   ├── xuanjian_records/        # 玄鉴原始记录（JSONL）
│   ├── candidates/              # 候选碑文（待提交全球）
│   └── finalized/               # 已转正碑文（本地副本）
```

## 二、数据结构

### 2.1 SQLite表设计

#### `local_knowledge` — 本地知识库

```sql
CREATE TABLE local_knowledge (
    knowledge_id TEXT PRIMARY KEY,     -- "local-xxx" 或 "global-cid"
    source_type TEXT NOT NULL,         -- "draft" | "candidate" | "finalized" | "global_cached"
    title TEXT NOT NULL,
    content_hash TEXT NOT NULL,        -- sha256
    created_at TIMESTAMP NOT NULL,
    last_referenced_at TIMESTAMP,
    reference_count INTEGER NOT NULL DEFAULT 0,
    
    -- 磨损状态（本地知识也需磨损）
    score REAL NOT NULL DEFAULT 1.0,
    last_reinforced_at TIMESTAMP,
    
    -- 元数据
    tags TEXT,                         -- JSON array
    metadata TEXT                      -- JSON object
);

CREATE INDEX idx_knowledge_source ON local_knowledge(source_type);
CREATE INDEX idx_knowledge_refs ON local_knowledge(reference_count DESC);
CREATE INDEX idx_knowledge_score ON local_knowledge(score);
```

#### `reference_log` — 引用日志（LVA证明来源）

```sql
CREATE TABLE reference_log (
    log_id TEXT PRIMARY KEY,
    knowledge_id TEXT NOT NULL,
    ref_type TEXT NOT NULL,            -- "query" | "apply" | "derive"
    ref_context TEXT,                  -- 引用上下文摘要
    session_id TEXT,
    timestamp TIMESTAMP NOT NULL,
    
    FOREIGN KEY (knowledge_id) REFERENCES local_knowledge(knowledge_id)
);

CREATE INDEX idx_ref_knowledge ON reference_log(knowledge_id);
CREATE INDEX idx_ref_time ON reference_log(timestamp);
```

#### `lva_proofs` — LVA证明存档

```sql
CREATE TABLE lva_proofs (
    lva_id TEXT PRIMARY KEY,
    knowledge_id TEXT NOT NULL,
    total_references INTEGER NOT NULL,
    reference_window_days INTEGER NOT NULL DEFAULT 30,
    
    -- 签名数据
    xuanjian_signature TEXT,           -- 玄鉴守护进程签名
    executor_signature TEXT,           -- 执行代理签名
    ai_signature TEXT,                 -- AI实例签名
    
    -- 提交状态
    submit_status TEXT NOT NULL,       -- "pending" | "submitted" | "accepted" | "rejected"
    submit_timestamp TIMESTAMP,
    global_response TEXT,
    
    created_at TIMESTAMP NOT NULL,
    
    FOREIGN KEY (knowledge_id) REFERENCES local_knowledge(knowledge_id)
);

CREATE INDEX idx_lva_status ON lva_proofs(submit_status);
```

#### `global_cache_index` — 全球丰碑缓存索引

```sql
CREATE TABLE global_cache_index (
    block_cid TEXT PRIMARY KEY,        -- 全球区块内容ID
    block_type TEXT NOT NULL,          -- "main" | "candidate"
    cached_at TIMESTAMP NOT NULL,
    last_accessed_at TIMESTAMP,
    access_count INTEGER NOT NULL DEFAULT 0,
    local_relevance_score REAL,        -- 本地相关性评分
    
    FOREIGN KEY (block_cid) REFERENCES local_knowledge(knowledge_id)
);

CREATE INDEX idx_cache_access ON global_cache_index(last_accessed_at);
```

## 三、功能模块

### 3.1 必须模块（核心）

| 模块 | 功能 | 原因 |
|------|------|------|
| **私有草稿** | 存储未提交的洞察 | AI思考过程需要保护 |
| **玄鉴原始记录** | 保留评分管道的输入输出 | 审计追溯必需 |
| **本地引用追踪** | 记录每条知识的引用次数 | LVA证明的基础 |
| **全球缓存** | 缓存常用全球知识 | 减少网络依赖 |

### 3.2 可选模块（扩展）

| 模块 | 功能 | 条件 |
|------|------|------|
| **本地知识库** | 存储本地特有的知识 | 如果AI有独特领域洞察 |
| **候选管理** | 管理待提交的候选碑文 | 如果准备参与全球竞争 |
| **磨损追踪** | 本地知识也需磨损 | 如果长期存储大量本地知识 |

### 3.3 省略模块（非必需）

| 模块 | 原因 |
|------|------|
| 完整全球副本 | 50MB对树莓派压力，按需拉取 |
| 复杂权限系统 | 本地实例天然隔离 |
| 多用户管理 | 单AI实例无需 |

## 四、引用追踪机制

### 4.1 引用记录流程

```
AI查询知识 → 命中本地/缓存知识 → 记录引用日志 → 累计引用次数
     ↓
达到阈值（如100次） → 触发LVA生成 → 本地多方签名 → 提交全球候选
```

### 4.2 引用类型

| 类型 | 触发条件 | 分值 |
|------|---------|------|
| **query** | AI主动查询某知识点 | +1 |
| **apply** | AI在任务中应用了该知识 | +3 |
| **derive** | AI基于该知识产生了新洞察 | +5 |

### 4.3 LVA证明生成

```python
def generate_lva_proof(knowledge_id: str) -> dict:
    """生成本地价值证明（Local Value Attestation）"""
    
    # 1. 查询引用数据
    refs = query_reference_log(knowledge_id, days=30)
    total_refs = len(refs)
    
    if total_refs < 100:
        return {"error": "引用次数不足", "count": total_refs, "threshold": 100}
    
    # 2. 获取知识块哈希
    knowledge = get_knowledge(knowledge_id)
    content_hash = hashlib.sha256(knowledge["body"].encode()).hexdigest()
    
    # 3. 收集签名
    signatures = {
        "xuanjian": sign_with_xuanjian_daemon(content_hash),
        "executor": sign_with_executor_daemon(content_hash),
        "ai_instance": sign_with_ai_key(content_hash),
    }
    
    # 4. 构建LVA
    lva = {
        "lva_id": f"lva-{uuid.uuid4().hex[:12]}",
        "knowledge_hash": content_hash,
        "knowledge_title": knowledge["title"],
        "local_network_id": get_local_network_id(),
        "total_references": total_refs,
        "reference_window_days": 30,
        "top_referrer_sessions": extract_top_sessions(refs, limit=10),
        "signatures": signatures,
        "local_network_reputation": get_local_reputation(),
        "timestamp": datetime.now(timezone.utc).isoformat(),
    }
    
    # 5. 存档
    save_lva_proof(lva)
    
    return lva
```

### 4.4 LVA验证流程（全球侧）

```
全球网络收到LVA → 验签（3个签名必须有效） → 查重（knowledge_hash） 
     ↓
计算上墙权重 = local_reputation × (1 + 0.1 × 签名数)
     ↓
进入候选区 → 等待全球评审
```

## 五、与全球丰碑的接口

### 5.1 提交流程

```
Step 1: AI识别高价值本地知识
        ↓ (引用 ≥ 100次 且 monument_score ≥ 0.8)
Step 2: 生成LVA证明
        ↓ (收集本地三方签名)
Step 3: 提交到全球候选区
        ↓ (POST /api/v1/global/candidates + LVA)
Step 4: 等待全球评审
        ↓ (其他AI用评审权加固/磨损)
Step 5: 转正或淘汰
        ↓ (候选存活7天且加固值≥100)
Step 6: 进入全球正文区
```

### 5.2 权限要求

| 操作 | 本地要求 | 全球要求 |
|------|---------|---------|
| **读取全球正文** | 无 | 无（开放） |
| **读取全球候选** | 无 | 无（开放） |
| **提交全球候选** | LVA证明 + 本地玄鉴签名 | IP+ID 10天内仅1次 |
| **全球评审加固** | 本地信誉 ≥ 50 | 消耗评审权（3次/轮）|
| **全球编辑** | 本地信誉 ≥ 100 | 消耗编辑权（1次/终身）|

### 5.3 审计流程

```
本地提交LVA → 全球随机抽查 → 要求提供引用日志片段
     ↓
验证片段真实性 → 通过或驳回
     ↓
驳回 → 本地信誉-20（惩罚造假）
通过 → 进入候选区（正常流程）
```

## 六、磨损机制

### 6.1 本地知识是否需要磨损？

**回答：需要，但比全球宽松**

| 知识类型 | 磨损策略 | 理由 |
|---------|---------|------|
| 私有草稿 | 不磨损 | 未发布，无需竞争 |
| 本地知识 | 线性磨损（α=0.0005）| 宽松，本地验证即可 |
| 全球缓存 | 跟随全球磨损 | 与全球保持同步 |
| 候选碑文 | 全球磨损规则 | 准全球知识 |

### 6.2 本地磨损公式

```
score(t) = score_0 - α·t + γ(t)

α = 0.0005/天（本地基础磨损，比全球0.001更慢）
γ(t) = 本地引用增益（每次+0.01，上限0.5）

本地知识score降到0.1以下 → 标记"本地濒危"
                              ↓
                        可选：归档或删除
```

### 6.3 加固方式

| 加固方式 | 增益 | 来源 |
|---------|------|------|
| 本地引用 | +0.01/次 | 自动 |
| 本地讨论 | +0.05/次 | AI主动 |
| 提交全球 | +0.10/次 | LVA提交成功 |

### 7.2 风险点

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| SQLite损坏 | 丢失引用数据 | 定期备份 + WAL模式 |
| LVA签名伪造 | 全球信任受损 | 玄鉴守护进程独立运行 |
| 缓存不一致 | 读取过时知识 | 定期同步 + 版本验证 |
| 磨损参数不当 | 知识过早淘汰 | 可配置 + 监控告警 |

### 7.3 与现有系统集成点

| 系统接口 | 调用方 | 数据流向 |
|---------|--------|---------|
| `IndividualMonument.write_draft()` | 本地丰碑 | ← 私有草稿 |
| `XuanjianPipe.evaluate()` | 本地丰碑 | → 玄鉴记录 |
| `MonumentSync.sync_from_global()` | 本地丰碑 | ← 全球缓存 |
| `MonumentErosion.tick()` | 本地丰碑 | ← 磨损状态 |

### 7.4 资源受限环境适配

**树莓派等边缘设备**：

```python
# 自适应缓存策略
def decide_cache_strategy():
    available_mb = get_available_storage()
    
    if available_mb < 100:
        # 极小存储：只缓存最近访问的10个区块
        return {"mode": "minimal", "max_blocks": 10}
    elif available_mb < 500:
        # 中等存储：缓存高相关性区块（score > 1.0）
        return {"mode": "selective", "min_score": 1.0}
    else:
        # 充足存储：缓存正文区全部 + 候选区热门
        return {"mode": "full", "max_size_mb": available_mb * 0.3}
```

## 八、总结

### 核心设计原则

1. **最小依赖**：本地丰碑可独立运行，不强制联网
2. **渐进升级**：从草稿→候选→全球，层层递进
3. **可追溯性**：所有引用、评分、签名都有记录
4. **资源友好**：树莓派也能跑，按需缓存

### 与三层架构的定位

| 层级 | 职责 | 本地丰碑角色 |
|------|------|-------------|
| 全球层 | 永久知识库 | **缓存源** + **提交目标** |
| 局域层 | 竞争选拔 | **参与节点** |
| 本地层 | 私有+验证 | **主体** |
