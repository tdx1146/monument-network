## 一、架构概览

```
调度器 (task_scheduler.py)
 │ 定时任务完成后写事件
 ▼
event_bus.jsonl ← 独立事件总线文件（7月16日新分离）
 │
 ▼
event_consumer.py（独立守护线程，3s轮询）
 │ 匹配 event_rules.yaml → 异步分派下游
 │ 限流 + 重试（指数退避3次）+ 死信保护
 ▼
下游动作（bridge/archive/escalate）
 │
 ▼
操作日志记录（operation_log.jsonl ← 加锁写入 log_writer.py）
```

## 三、event_schema.yaml（数据契约）

```yaml
version: "1.0"
event_types: [task_complete, anomaly, audit_result, milestone, consumer_action]
required_fields: [t(ISO8601), event_type, producer, result(OK/FAIL/WARN/TIMEOUT)]
```

所有写入 `event_bus.jsonl` 的事件必须遵循此格式。

---

## 四、事件生命周期（6站链）

```
① 调度器 run_task() → 写 JSONL 到 event_bus.jsonl ✅
② event_consumer poll_loop() → 读 event_bus.jsonl ✅
③ event_consumer _dispatch() → subprocess 调下游 ✅ (shell白名单+黑名单)
④ event_consumer 写消费记录 → operation_log.jsonl ✅ (加锁)
⑤ 玄鉴 _audit_event_chain() → 读 event_bus.jsonl ✅ (风暴/失败率/死信/循环检测)
⑥ 玄鉴写审计结果 → daemon_audit.log ✅
```
