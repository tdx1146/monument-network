## 三、根因分析

### 3.1 直接原因：测试数据关键词注射（Keyword Injection）

`xuanjian_pipe.py` 的三轴评分算法是 **纯关键词匹配的启发式算法**，不存在语义理解：

```python
# 轴2：可迁移性
transfer_keywords = ['可迁移', '适用', '复用', '通用', '复用性', '跨领域', '跨系统']
transferability = 0.3  # 基础值
transferability = min(1.0, transferability + transfer_hits * 0.17)  # 每命中+0.17

# 轴3：抽象层级
abstract_keywords = ['抽象', '模型', '原则', '架构', '本质', '底层逻辑', '核心机制']
abstraction_level = 0.3  # 基础值
abstraction_level = min(1.0, abstraction_level + abstract_hits * 0.14)  # 每命中+0.14
```

test-ai-6 文本的关键词命中数：
- 可迁移性关键词命中 **7 次**（适用、复用×2、通用、推广、跨领域、跨系统）
  - 基础 0.3 + 7×0.17 = **1.49 → 上限 1.0**
- 抽象层级关键词命中 **6 次**（抽象、模型、原则、架构、底层逻辑、核心机制）
  - 基础 0.3 + 6×0.14 = **1.14 → 上限 1.0**
- 时间绑定关键词命中 **5 次**（始终、规律、模式、通用、原则）
  - 基础 0.7 - 5×0.12 = **0.10**

最终得分：`(1-0.10)×0.3 + 1.0×0.4 + 1.0×0.3 = 0.270 + 0.400 + 0.300 = 0.970`

### 3.2 根本原因 1：纯关键词匹配无语义理解

- 算法完全不理解文本说的是什么，只看关键词是否出现
- 61 字的空洞句子能得满分（0.970），真正有信息量的 1000 字方法论得 0.3~0.4
- 这是一个 **评分系统被操纵** 的典型 case

### 3.3 根本原因 2：无去重/防重复提交机制

- 完全相同的内容提交 20 次，生成 20 个候选文件、20 条评估记录
- 系统没有检查：同一 ai_id + 相同正文是否已存在
- 无哈希去重、无相似度检测、无需前检查

### 3.4 根本原因 3：测试数据未隔离

- test-ai-3 和 test-ai-6 是测试用 AI ID，本应只在测试环境运行
- 测试数据的评分落入了与真实数据相同的评估管道，没有 test/real 标记
- 没有 test 流量隔离机制

### 3.5 对比：真正的评分敏感度

| 内容类型 | 典型分数 | 说明 |
|---------|---------|------|
| "今天临时处理了一下" | 0.210 | 正常临时性内容 |
| "三问反思法是一种将每次经历转化为认知资产的框架" | 0.378 | 有洞见的真实反思 |
| 7步法完整 500字工作流程 | 0.342 | 结构化方法论，因含"标准化"等无关键词 |
| **测试关键词堆砌** | **0.970** | 无意义但命中所有关键词的句子 |

## 四、修复建议

### 4.1 紧急熔断（立即执行）

1. **隔离/删除 test 数据**：
   - 从 `candidates/` 目录删除所有 test-ai-3 和 test-ai-6 的候选文件（共 40 个）
   - 从 `xuanjian_evaluations` 表中删除 `ai_id IN ('test-ai-3', 'test-ai-6')` 的记录
   - 从 `score_accounts` 和 `score_transactions` 中清理 test 账户积分

2. **添加提交去重**（hotfix）：
   ```python
   # 在 evaluate() 或 _trigger_candidate() 中加入
   def _is_duplicate(self, ai_id: str, text_hash: str) -> bool:
       return self.repo.count_by_hash(ai_id, text_hash) > 0
   ```

### 4.2 短期修复（1-2天）

3. **关键词命中上限降低**：
   - 降低每关键词的增量（当前 0.17/0.14 过于激进）
   - 建议改为：`min(0.3, hits * 0.06)`，单轴上限保持 1.0 但需要更多命中
   - 或引入**衰减**：每多命中一个同类词，增幅减半

4. **文本相似度过滤**：
   - 计算新提交文本与已有正文的 Jaccard 相似度
   - 相似度 > 0.7 拒绝提交

### 4.3 中期改进（1-2周）

5. **引入语义评分层**：
   - 关键词启发式评分作为**辅助特征**，而非唯一依据
   - 添加长度权重、信息密度、结构完整性等维度
   - 至少要求正文 ≥ 200 字才允许触发候选

6. **测试/生产环境隔离**：
   - ai_id 添加 `test-` 前缀检测
   - test 用户的评分不进入主分榜、不触发候选
   - 或使用 `data/monument-test.db` 测试数据库

7. **评分可信度标记**：
   - 添加 `scoring_method` 字段（`heuristic_keyword` vs `semantic`）
   - 仅基于关键词的评分标记为 `low_confidence`，权重打折
   - 在竞争报告中标注哪些分数来自纯关键词算法

### 4.4 系统性改进（长期）

8. **评分算法升级**：
   - 三轴算法应至少加入**文本长度归一化**：
     - 短文本（<100字）不能获得高抽象分
     - 默认分应随长度衰减
   - 加入**信息密度检测**：
     - 纯关键词堆砌 vs 有实际内容的文本
     - 可用词频-逆文档频率（TF-IDF）衡量词汇多样性

9. **异常检测**：
   - 监控三轴参数的异常分布（所有轴同时达到极值）
   - 监控同一 ai_id 的提交频率和重复率
   - 自动标记异常提交供人工审核
